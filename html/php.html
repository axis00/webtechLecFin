About PHP

Hypertext Preprocessor is now popularly called as PHP. PHP is created by Rasmus Leordorf in 1994 and it was released to the public in June of 1995. It is an open source scripting language that is primarily used for web applications. At first, this scripting language was originally used by Leordorf to track his online resume. That is why it is coined as "Personal Home Page Tools" or "PHP Tools". Later on, Leordorf expanded the capabilities of this language

PHP can be embedded into HTML documents.

Version History

  • PHP/FI or PHP 2.0

    This version of the language is combined together with the Form Interpreter (FI) that was also developed by Rasmus Leordorf.

  • PHP/FI 2
  • PHP 3

    • Announced and released on June 1998. It comes with extensibility feature to provide users an interface for multiple databases, protocols and APIs. It also supports object-oriented programming and has a more powerful and consisten language syntax.

  • PHP 4

    • This version of PHP was officially released in May of 2000. PHP 4 support more web servers, HTTP sessions, output buffering and more security for handling user input.

  • PHP 5

    • The latest version of PHP that was released in July of 2004.

File Upload Handling

To handle file uploads, the use of the form tag in the HTML file is needed. One needs to specify the action attribute, which is a PHP file, and the method, which should be POST. An 'enctype' attibute should also be specified in the form tag. The attribute should be set to "multipart/form-data", this allows different file uploads.

In the PHP file, the '$_FILES' global variable will be used inorder to store the uploaded file information.

The function 'move_uploaded_file(file,newloc)' moves the uploaded file (file) to a new location (newloc).

Session Handling

Session handling allows the server to store user information used across the multiple pages of a web application. These include the requests made. Usually, sessions are implemented in web applications that require the user to login. These sessions last until the user closes the browser or logs out from the web application.

The 'session_start()' function starts the tracking of the session. Session variables could also be set using the '$_SESSION' global variable. '$_SESSION[]' put the variable name inside the square brackets. These variables are only avaialable while the session is active.
